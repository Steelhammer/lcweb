lcapi specification - v1

The lcapi is a REST API, to control the lightcontrol command line tool through the web.

Commands are sent using the resources, http verbs and JSON data.

Resource receiver

Resource GET PUT POST DELETE
Collection URI, such as http://site/lcweb/lcapi/v1/receiver/ List the receivers as an array of JSON objects. N/A Create a new receiver based of the supplied JSON object. Returns the created receiver as a JSON object. N/A
Element URI, such as http://site/lcweb/lcapi/v1/receiver/12 Retrieve a JSON representation of the addressed receiver. Update the addressed receiver based of the supplied JSON object. Returns the updated receiver as a JSON object. N/A Delete the addressed receiver.

Receiver attributes

* id       - id of the receiver in the database
* name     - Logical name given by the user, e.g. 'Bedroom - window'
* house    - Internal id to connect to the receiver HW
* unit     - Internal id to connect to the receiver HW
* isdimmer - Reciver HW supports dimming
* status   - On/Off/Dimmed: <level>
* learned  - Receiver setting is connected to at least one receiver HW
* level    - Dimmer level (0-15)

Receiver JSON format

[{
    "id": "1",
    "name": "Name A",
    "house": "352228",
    "unit": "11",
    "isdimmer": "0",
    "status": "Off",
    "learned": "1"
}]

Create receiver JSON format

[{
    "name": "Name B",
    "isdimmer": "1"
}]

Update receiver JSON format

Edit receiver settings

[{
    "name": "Name C",
    "isdimmer": "0"
}]

Activate/Deactivate receiver

[{
    "command": "on" //off
}]

Dim receiver

[{
    "command": "dim",
    "level" : 8
}]

Send learn command to receiver HW

[{
    "command": "learn"
}]

Resource timer

Resource GET PUT POST DELETE
Collection URI, such as http://site/lcweb/lcapi/v1/timer/ List all timers as an array of JSON objects. N/A Create a new timer. N/A
Element URI, such as http://site/lcweb/lcapi/v1/timer/18 Retrieve the timers associated with the addressed reveicer as an array of JSON objects. Update the addressed timer. N/A Delete the addressed timer.

Timer attributes

* id       - id of the timer in the database
* recid    - id of the receiver affected by the timer
* ontime   - time to activate the receiver
             Formats accepted:
               12:00 (fixed time)
               12:00R30 (12:00+Random(30) minutes, i.e. 12:01-12:30)
               SUNSET (Sunset time calculated by calendar and coordinates)
               SUNRISE
               SUNSET+30 (30 minutes after sunset time)
               SUNSET-60 (1 hour before sundet time)
* offtime  - time to deactivate the receiver
             Formats, see ontime
* days     - days of the week the timer is active e.g. 1234567 (all days), 67 (Saturday and Sunday) 
* dimlevel - dimmer level to use when activating the timer (0-15)

Timer JSON format

[{
    "id": "5",
    "recid": "1",
    "ontime": "20:00",
    "offtime": "23:10",
    "days": "1234567",
    "dimlevel": "6"
}]
